Class Watir::Image
In: watir.rb
Parent: ObjectActions

This class is the means of accessing an image on a page. It would not normally be used by users, as the image method of IEController would return an initialised instance of an image. many of the methods available to this object are inherited from the ObjectActions class

Methods

fileCreatedDate   fileSize   hasLoaded?   height   highLight   new   save   src   to_s   type   value   width  

Public Class methods

new( ieController, how , what )

Returns an initialized instance of a image object 

  * ieController  - an instance of an IEController
  * how         - symbol - how we access the image
  * what         - what we use to access the image, name, src, index, id or alt

Public Instance methods

fileCreatedDate()

this method returns the file created date of the image

fileSize()

this method returns the filesize of the image

hasLoaded?()

This method attempts to find out if the image was actually loaded by the web browser. If the image was not loaded, the browser is unable to determine some of the properties. We look for these missing properties to see if the image is really there or not. If the Disk cache is full ( tools menu -> Internet options -> Temporary Internet Files) , it may produce incorrect responses.

height()

returns the height in pixels of the image, as a string

highLight( setOrClear )

this method highlights the image ( in fact it adds or removes a border around the image) 

 * setOrClear   - symbol - :set to set the border, :clear to remove it
save(path)

This method saves the image to the file path that is given. If the image src references a local url (file:///…) then the image is saved directly from the src. If the image references any other kind of url then the ‘Temporary Internet Files’ cache is searched for the image. Raises a CacheItemNotFound exception if the image cannot be located path - directory path and file name of where image should be saved

src()

this method returns the source url of the image

to_s()

returns a string representation of the object

type()

returns the type of the object - ‘image’

value()

a value doesnt exist for an image, so this will return an empty string

width()

returns the width in pixels of the image, as a string
