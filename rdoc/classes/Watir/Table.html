Class Watir::Table
In: watir.rb
Parent: ObjectActions

This class is used for dealing with tables. This will not be normally used by users, as the table method of IEController would return an initialised instance of a table. many of the methods available to this object are inherited from the ObjectActions class

column_count(index=1)

This method returns the number of columns in a row of the table. Raises an UnknownTableException if the table doesn’t exist. 

  * index         - the index of the row
column_values(columnnumber)

Returns an array containing all the text values in the specified column Raises an UnknownCellException if the specified does not exist in every row of the table 

  * columnnumber  - column index to extract values from
each() {|TableRow.new(@ieController ,:direct, row(i) ) }| ...}

iterates through the rows in the table. Yields a TableRow object

highLight(setOrClear )

override the highlight method, as if the tables rows are set to have a background color, this will override the table background color, and the normal flsh method wont work

name()

tables dont have name, so return an empty string

row_count()

This method returns the number of rows in the table. Raises an UnknownTableException if the table doesnt exist.

to_a()

This method returns the table as a 2 dimensional array. Dont expect too much if there are nested tables, colspan etc. Raises an UnknownTableException if the table doesn’t exist.

to_s()

returns the properties of the object in a string raises an ObjectNotFound exception if the object cannot be found

value()

tables dont have value, so return an empty string
