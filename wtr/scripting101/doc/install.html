Scripting Web Tests

Installation Instructions

These instructions describe how to install the software we’ll use in our Scripting Web Tests class (aka Scripting for Testers).

We will be using Ruby (a programming language) and WATIR (a library) on your machine to test server software. You can run the server software on your computer.

Hardware Requirements

The class software runs on Windows machines. We recommend Windows 2000 or Windows XP. You should have Administrator privileges.

We believe that it will also work on Windows NT and Windows 98, but we don’t support these in our classes. (Our experience has been that machines that are still running these operating systems often are too slow or have other problems that we don’t have time for in a class environment. Sorry.)

The WATIR software requires Internet Explorer and therefore only runs on Windows. The Timeclock server was developed on a Macintosh, and should run on any machine supporting Ruby (Unix, Linux, Windows). (The Timeclock “product tests” will not run on Windows using the mswin build—the one you use for the class. They will run on Windows using the cygwin build.)

Installing the Software

All of the files you need are included on the class CD. For your convenience we have also provided links to where these materials can be found online.

Ruby

We are currently using this Ruby installer: ruby181-12.exe. (There are known problems with ruby 1.8.1-13; don’t use it.)

  1. Double-click this exe file to install.
  2. Select the default installation options (choose all options; install in c:/ruby).
  3. If you are installing as Administrator you don’t need to restart windows. Otherwise, you should.

WATIR Library

This library is hosted on rubyforge.org: watir.tar.gz

  1. Extract the contents of this archive into your hard drive (c:). A “watir” directory will be created automatically.

Scripting Class Materials.

You will also need the materials hosted on rubyforge.org: scripting101.tar.gz

  1. Extract the contents of this archive into your hard drive (c:). A “scripting101” directory will be created automatically.

Running the Timeclock Server.

You probably won’t need to do this for class, since you can use your instructor’s server. Otherwise, here is how to start up a timeclock server on your machine.

  1. Start a command shell. (Start-> Run, enter “cmd”.)
  2. Enter “cd c:\scripting101\timeclock\bin”.
  3. Enter “ruby http-server.rb”. The server is now running.

Note: If you close this command window, the server will stop.

You can verify that the server is running correctly by running some sample tests.

  1. Start a new command shell.
  2. Enter “ruby c:\scripting101\solutions\suite.rb -b”.

Note: These tests will run unbearably slowly if Internet Explorer is configured to use a proxy server that is unavailable.
