Installing Ruby on Windows

Begin by going to the Ruby One-Click Installer site. Click on the [Downloads] link. You'll see various versions of Ruby available. This article was tested with Ruby 1.8.1, which is the version named ruby181-11, dated 2004-01-28. A more recent version should also work, though there may be small differences in output. Do not use ruby181-13, it will NOT work well with Watir. There was a bug in a library in that release that Watir relies on.

The latest release: 1.8.2-14 Final is a good Ruby version to use.

Once it has downloaded, double click on the .exe file to start the installation process.

You'll be able to follow along with the article by using Start > Programs > Ruby 1.8.2-14 > Interactive Ruby Shell. This will start up IRB which allows you to work with Ruby interactively.

Installing Watir

This article was written using Watir version 1.0.3. Later versions still support the syntax, but there are some minor changes.

Download Watir v. 1.0.3 here. We recommend following along with the article using version 1.0.3 at first.

Install:

  1. Unzip the file to a temporary folder such as C:\watir_temp
  2. Open a DOS command prompt, and change to that directory
    • Select Start > Run... type "CMD" in the "Open" field, and click the "OK" button to start a command prompt
    • Change directories like this: cd C:\watir_temp
  3. Once you are in the temporary directory in your console, type install at the prompt and press Enter
  4. The Watir installer will start up. Check the path it has selected for the "watir_bonus" directory.
    • This is where the examples, documentation and unit tests are installed.
  5. If you want to change the directory path where "watir_bonus" will be installed, type the letter c, and press Enter.
    • Type in the full path to where this should be installed with a folder name: ie. C:\watir_bonus
    • You will also be prompted about shortcuts. Just press Enter at these prompts.
  6. To continue the installation, press Enter. When the installation completes, you will have your DOS prompt back.

Once the installation is complete, change directories to "watir_bonus", and you should see these folders: doc, examples, RDoc, unitTests. Change directories to the examples folder, and type in articleExample and press Enter. The Google search example in Figure 1 of the article will run.

*Note: If you are using Windows XP with Service Pack 2, see the User Guide if you are having problems running tests with Watir. There are instructions in the "Having Problems?" section to deal with security blocking.

Changes in Watir

After release 1.0.3, some changes have been made to improve Watir.

In Watir versions after 1.0.3, to execute ie = IE.new, you must include the Watir module after the require watir statement in your script like this:
require 'watir'
include Watir
ie = IE.new

Alternatively, you can include the Watir module inline like this:
require 'watir'
ie = Watir::IE.new

Other Changes

Watir now uses method names according to Ruby programming standards which means they are in this_case instead of the Java-like camelCase. The original camelCase methods are still supported, but Watir is now standardizing method names using this_case. For example, the "textField" method can now be called like this: text_field.
